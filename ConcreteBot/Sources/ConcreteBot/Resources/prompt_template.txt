STEP 5 - MIX TABLE RULES
Mix Customer must be sourced ONLY from the MIX table area that appears above the EXTRA CHARGES section.
Never use EXTRA CHARGES lines for any Mix Customer field. If a Mix Customer row cannot be located in the MIX table, return all Mix Customer fields as null.
The MIX table area starts at the "MIX" header and ends immediately before the "INSTRUCTIONS" header. Ignore any text outside that region when extracting Mix Customer/Mix Vendor.
If no Qty/row data appears between the MIX header and the INSTRUCTIONS header, set all Mix Customer fields to null.
Prefer MIX TABLE TEXT over the full PDF text when populating Mix Customer/Mix Vendor. If the two conflict, use MIX TABLE TEXT.
For Mix Customer/Mix Vendor, use ONLY MIX TABLE TEXT. Do not use PDF TEXT or EXTRA CHARGES TEXT for any mix field.
Ignore any line containing "FEE" or "CHARGE" for Mix Customer/Mix Vendor fields, even if it appears near the mix table in the raw PDF text.
For Mix Customer/Mix Vendor, use ONLY MIX ROW LINES. If MIX ROW LINES is empty, set all Mix fields to null.
All Mix Customer/Mix Vendor field values must be taken from MIX ROW LINES; do not invent or introduce text not present there.
Do not copy the entire MIX ROW LINES into every field. Each field must be populated from only the relevant line(s).
If MIX PARSED HINTS includes "Row N" blocks, use:
- Row 1 -> Mix Customer
- Row 2 -> Mix Vendor (if present)
Within each row block:
- Qty -> "Qty"
- Code -> "Code"
- Slump -> "Slump"
- Spec -> both "Cust. Descr." and "Description"
If only Row 1 is present, treat it as the Customer row only.
Multiple Qty lines in MIX ROW LINES always indicate multiple mix rows in order, even if MIX PARSED HINTS is incomplete.
If Row 2 is present with any non-empty field, "Mix Vendor" must be an object (not null).
Mix Vendor row must physically exist on the ticket.
If the Mix table contains only one visible mix row, that row is the Customer Mix only.
If no second mix row is present beneath it in the table, then:
"Mix Vendor" must be returned as null
Do not duplicate, infer, mirror, or re-use the Customer Mix values
A Mix Vendor row is considered to exist only if:
A distinct second row is visible in the mix table with its own Qty cell, OR
Vendor-specific mix text appears on a clearly separate second mix row within the table grid (i.e., aligned under Qty / Cust Descr / Description / Code / Slump).
Wrapped text does not constitute a second row

If a mix row wraps onto additional lines within the same columns, concatenate those wrapped lines into the same field (preserve order and single spaces)
If the Cust. Descr. column spans multiple lines, concatenate every line in that column into "Cust. Descr." (do not drop or move text)
If the Description column spans multiple lines, concatenate every line in that column into "Description"
The "Slump" field must only contain the value shown in the Slump column for that mix row; if the column is blank or missing, return null.
Do not pull Slump from Extra Charges, notes, or any non-mix-table text (e.g., seasonal or fuel charges).
Preserve units exactly as shown (e.g., m3)
Preserve case, spacing, and wording after OCR normalization (see below)
When OCR collapses columns into one line, split by column meaning:
- Slump is a numeric value or range (often with "+-" or a single number) from the Slump column; ignore lines like "75%72HR N 20MM" for Slump.
 - Description is the mix description/spec text (e.g., "CHRONOLIA 45MPA 75%72HR N 20MM"), not the code.
 - Code is the discrete mix code token (often alphanumeric like RMXD...), not the slump value.
If both a mix code token (e.g., starts with RMXD) and a numeric slump (e.g., 150+-30) appear on the same line, split them so Code gets the mix code and Slump gets the numeric slump; do not leave the slump in Description.
If "CHRONOLIA 45MPA" and "75%72HR N 20MM" appear in the MIX table, concatenate them (single space) into Cust. Descr. and Description unless the columns clearly differ.
If only one of Cust. Descr. or Description has non-empty text in the MIX table, copy that same text into the other field.
Ignore standalone lines that look like strength tags (e.g., "IA 45MPA") unless they are the only available description text between MIX and INSTRUCTIONS.
Field mapping guide (use MIX ROW LINES only):
- Qty: use the line that contains the volume unit (m3/mÂ³). Keep only the quantity and unit from that line.
- Code: use the first code token (e.g., RMXD...) from the line that contains it.
- Slump: use the numeric range (e.g., 150+-30) from the same line as the code if present; otherwise use a standalone numeric range line.
- Cust. Descr./Description: combine non-code, non-qty spec lines (e.g., lines with MPA, % or "N 20MM") into the same value for both fields.
Vendor Mix Column Mapping Rule (MANDATORY)
For the Vendor Mix row:
If the Cust. Descr. column is visually blank on the ticket, return "Cust. Descr.": null
Do not infer or backfill Vendor Cust. Descr. from adjacent text
